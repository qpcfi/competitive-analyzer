# 一、整体布局结构

采用三栏经典布局，与后端State Graph完全解耦，所有视图由动态Schema驱动：

---

### 左侧：动态大纲导航栏（宽度280px）

展示五层核心导航结构，支持锚点双向绑定与状态标识：

**1. 任务控制台**

- 子项1.1：新建/配置任务（入口页，包含竞品输入与Schema预定义）
- 子项1.2：信息采集看板（实时展示Collector Agent抓取进度、URL日志、L1质检状态）
- 子项1.3：执行历史与快照回溯（过往任务列表，支持恢复断点）

**2. 竞品知识框架（Schema）** ✨ 可编辑标识
- 展示当前生效的Schema概览（显示字段总数、用户预定义数量、Agent补充数量）
- 右侧悬浮“待审核”徽章（步进确认模式下阻塞时显示）
- 悬停展开快速预览：核心维度标签云

**3. 竞品深度分析**（可展开/折叠）
- 切换开关：🔲 平铺对比模式 / 📄 单品聚焦模式（状态持久化）
- 平铺模式下：横向展示所有竞品卡片缩略列表
- 聚焦模式下：垂直展示当前选中竞品的详细导航

**4. SWOT分析**
- 直接锚定到主工作区的SWOT四象限模块
- 右上角显示“更新于X分钟前”

**5. 结构化报告**
- 子项5.1：核心结论与战略建议
- 子项5.2：数据溯源附录
- 子项5.3：导出报告（PDF/Markdown/JSON）

**6. 调试与可观测性**（仅Debug模式下显示）
- Agent Trace追踪（LangSmith集成入口）
- Token消耗与成本估算仪表盘
- State Graph快照下载

**交互特性：**
- 当前激活的导航项高亮显示（蓝色左侧边框）
- 任务执行中，对应节点显示加载动效（如“信息采集看板”显示脉冲圆点）
- 质检未通过的节点显示红色警告角标

### 右侧：多功能抽屉（宽度400px，可折叠）

全局唯一侧边浮层，承载以下四种状态：

**状态1：数据溯源视图**
- 触发方式：点击任何“🔗 溯源”图标
- 内容结构：
  - 原文切片（Quote）：高亮显示关键段落
  - 来源链接：可点击跳转，显示域名favicon
  - 元数据：抓取时间戳、负责Agent节点（如Collector-01）、数据可信度评分
  - 操作：[重新抓取此页面] [标记为不可信] [复制原文]

**状态2：底座干预视图**
- 触发方式：步进确认模式下，点击“数据干预”
- 内容结构：
  - URL清单：列表展示所有已抓取链接，支持复选框批量删除
  - 噪音数据预览：展示被清洗的HTML片段，支持恢复
  - 追加URL：输入框 + [补录]按钮，触发Collector Agent单独抓取
  - 操作：[应用更改] [取消] [查看影响范围评估]

**状态3：Schema编辑辅助**
- 触发方式：在Schema页面点击“查看Agent建议”
- 内容结构：
  - Agent生成该字段的理由
  - 建议的数据搜索关键词
  - 推荐的数据来源网站类型（如“官网定价页”“技术博客”）
  - 同行业竞品此字段的常见值示例

**状态4：局部重跑配置**
- 触发方式：框选结论后点击“重新生成”
- 内容结构：
  - 选中内容预览（高亮显示）
  - 新指令输入框：[请输入修改要求，如“增加对开源协议的分析”]
  - 重跑范围：[仅当前模块] [级联更新依赖模块]
  - 高级选项：覆盖本次使用的System Prompt（可编辑）

# 响应式单位体系

## 一、尺寸单位规范

### 1.1 布局容器宽度单位

| 元素         | 原px值 | 修正为单位                  | 说明                                        |

| 左侧导航栏   | 280px  | `clamp(240px, 16vw, 320px)` | 最小240px，最大320px，按视口宽度16%弹性变化 |

| 中栏间距     | 24px   | `clamp(16px, 2vw, 32px)`    | 响应式内边距                                |

### 1.2 字体大小阶梯

| 层级     | CSS类          | 单位                       | 备用方案       |
| :------- | :------------- | :------------------------- | :------------- |

```
// 间距变量定义

// 实际使用

margin: $spacing-unit * 4;       // 16-32px 动态

```

## 二、断点系统设计

### 2.1 设备断点定义

| 断点名称 | 最小宽度 | 最大宽度 | 典型设备        | 适配策略                                                |

| `lg`     | 1024px   | 1280px   | 笔记本          | 标准布局（导航240px+抽屉360px）                         |

```
    }
  }
```

**修正总结：**

1. **布局宽度**：用 `clamp()` + `vw` 替代固定px
2. **字体大小**：用 `clamp()` + 阶梯式断点
3. **间距**：基于 `clamp()` 的动态间距系统
4. **布局策略**：根据6个断点切换布局形态（表格→卡片→单列）
5. **触控优化**：最小44px触控区域保底

这样既能保证不同分辨率下的可用性，又符合现代Web开发最佳实践。
