# 一、 核心编排与 AI 模型层

- **Agent 编排框架**：`LangGraph` (Python 版本)。用于构建 Orchestrator 中枢调度、实现 State Graph 循环状态流转与 Checkpoint 断点持久化。
- **大语言模型**：豆包大模型 (Doubao API)。负责意图识别、复杂长文本理解、SWOT 推理与 Function Calling 结构化输出。

# 二、 后端服务层

- **核心 Web 框架**：`FastAPI`。原生支持异步并发，利用 Pydantic 进行严格的数据契约与 Schema 校验。
- **实时通信协议**：`Server-Sent Events (SSE)`。结合 FastAPI 的 `StreamingResponse`，向前端流式单向推送 Agent 运行状态、日志数据与阶段性产物。
- **数据采集与处理**：
  - `Playwright`：处理 SPA/动态渲染网页的抓取。
  - `BeautifulSoup`：执行 DOM 树的文本提取与清洗。
  - `LangChain Text Splitters`：执行超长研报与长文本的 Chunking 切片。

# 三、 数据存储与可观测性层

- **主数据库引擎**：`PostgreSQL`。深度依赖 `JSONB` 字段类型，用于存储动态生成的竞品 Schema、带有溯源 URL 的底座素材及运行状态快照。
- **可观测性与 Trace 追踪**：引入 `LangSmith` 等组件。用于收集和管理 Agent Trace，实现多 Agent 执行过程、上下文传递细节全链路可查，确保信息溯源与报告生成的权威性 。 

# 四、 前端交互与视图层

- **核心框架**：`Next.js` (React)。利用内置的 API Routes 作为 BFF 层，封装大模型直连请求，处理 SSE 订阅。
- **UI 组件库**：`Arco Design` 或 `Ant Design`。构建前端展示页面，保障汇报演示时的流畅交互体验 。  
- **可视化引擎**：`ECharts` 或 `AntV` (业务图表)；`React Flow` (工作流大盘，展示 DAG 节点)。
